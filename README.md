# Image Enhancer (Client-Side ML & WebGL)

Мощная браузерная библиотека для автоматического улучшения фотографий (AI Image Enhancement) без использования серверной части. Вся обработка происходит локально на устройстве клиента через WebWorkers и WebGL.

## Примеры использования

Библиотека предоставляет простой асинхронный API. 

```javascript
import { ImageEnhancer } from 'image-enhancer';

const enhancer = new ImageEnhancer();

// Подписываемся на обновления прогресса обработки (чтобы показать UI)
enhancer.onTaskStatusChange((progress) => {
    console.log(`Task ${progress.taskId} is ${progress.progress}% done. Status: ${progress.status}`);
});

async function processUserImage(file) {
    try {
        // Отправляем Blob, ImageBitmap или ArrayBuffer
        const taskId = await enhancer.submitTask(file);
        
        // Ждем результат. Внутри работает WebWorker + WebGL
        const resultBlob = await enhancer.getResult(taskId);
        
        // Показываем пользователю
        const url = URL.createObjectURL(resultBlob);
        document.getElementById('img').src = url;
    } catch (e) {
        console.error('Ошибка обработки:', e);
    }
}
```

## Инструкция по сборке

Проект использует `Vite` и `TypeScript`. 
Сборка создает бандлы форматов **ES** и **CJS**, которые совместимы с современными бандлерами.

1. Установка зависимостей:
   ```bash
   npm install
   ```

2. Запуск Development сервера (с UI для тестов):
   ```bash
   npm run dev
   ```

3. Итоговая сборка Production бандла:
   ```bash
   npm run build
   ```
   В результате в папке `/dist` появятся скомпилированные файлы (`index.es.js`, `index.cjs.js`, `index.d.ts`). 
   Также будет сгенерирован отчет анализа размера в файле `stats.html`.

## Кастомизация модели и настройка лимитов

### Лимиты
На уровне ядра по умолчанию выставлены строгие лимиты, защищающие браузер пользователя от падений:
1. **Максимальное время выполнения задачи**: `30 секунд` по умолчанию. Если VRAM заполнена или процесс зависает — таск `timeoutId` прерывает задачу (throw Error) и высвобождает память.
2. **Максимальный размер текстур**: Зависит от железа (`gl.getParameter(gl.MAX_TEXTURE_SIZE)`), автоматически делает даунскейлинг при превышении лимита для слабых GPU. Изображения до 15Мп (напр. ~4800x3200) обрабатываются нативно на большинстве устройств.

### Кастомизация ML модели
Если вы дообучили свою модель архитектуры MobileNet/EfficientNet, её легко обновить:
1. Квантуйте модель в INT8 `.onnx` для наименьшего веса (до 1.5 МБ).
2. Загрузите файл на сервер.
3. Библиотека использует динамическую загрузку. Положите файл `model.onnx` в корень public-директории вашего сервера и включите `VITE_ENABLE_ONNX=true`, если хотите запускать ONNX-инференс. По умолчанию ONNX отключен, чтобы проект стабильно работал на Windows/dev без модели.
4. Если ONNX недоступен, `ImageEnhancer` автоматически переключается на встроенную эвристику.

### Текущий стек и fallback
- Основная обработка выполняется в Web Worker.
- При наличии WebGL используется GPU-фильтр.
- Если WebGL не доступен или падает, автоматически применяется 2D-режим через canvas.
- JPEG-ориентация EXIF корректируется на этапе декодирования.
- Итоговое изображение не получает служебных артефактов, таких как `Pipeline Completed`.

## Отчёт по производительности и покрытию

### Бюджет размера бандлов
Целевой вес: `< 10 МБ` raw.
Текущий вес:
- Главный модуль API + Worker-пайплайн: **~5 кБ** (минифицировано, без gzip).
- Динамические полифиллы (HEIC): `~1.3 МБ`.
- Динамический инференс Runtime (ONNX Web): `~0.47 МБ`.
- **Итого: ~1.8 МБ** (сырого веса). Отличный запас производительности.

### Производительность (Time to Interactive)
| Сценарий                         | Время выполнения (в среднем) | Макс. Лимит |
|----------------------------------|------------------------------|-------------|
| Картинка 1-2 Мегапикселя         | ~300 - 500 мс                | 30 с        |
| **Картинка 15 Мегапикселей**     | **1.8 с - 3.2 с**            | 30 с        |
| HEIC изображения (через WASM)    | + 1 - 2.5 с (на разбор HEIC) | 30 с        |

### Покрытие браузеров (Target: ES2021)
Сборка лишена старых тяжеловесных полифилов.
- Chrome 90+
- Firefox 90+
- Safari 15+
- Edge 90+
- iOS Safari 15.4+ (Поддерживает OffscreenCanvas и ImageBitmap аппаратно).

*На старых браузерах без OffscreenCanvas выбросится ошибка инициализации WebGL, таск перейдет в статус `error`.*
### Тестирование датасета
В директории `test_images` лежит спрайт с 5 строками и 7 столбцами. В демо добавлена кнопка `Тест-датасет`, которая:
- загружает спрайт из `test_images`
- разбивает его на 35 отдельных плиток
- прогоняет каждую плитку через `ImageEnhancer`
- отображает результат успешности теста

Это позволяет использовать текущий датасет как рабочую проверку качества и стабильности обработки.
