## 📊 Реализованные этапы обработки

```
1. Декодирование (0-20%)         → decodeImage()
2. Анализ / ML-инференс (20-40%) → runMLInference()
3. Обработка WebGL (40-90%)      → processWebGL()
4. Кодирование результата (90-100%) → encodeResult()
```

**ML-параметры:**
- `brightness` (яркость): -1 .. 1
- `contrast` (контраст): 0.5 .. 2.0  
- `saturation` (насыщенность): 0 .. 2.0

## 📦 Public API (полностью соответствует заданию)

```typescript
interface ImageEnhancerAPI {
  submitTask(image: Blob | ArrayBuffer | ImageBitmap): Promise<string>;  // → taskId
  getTaskStatus(taskId: string): Promise<TaskProgress>;                    // статус + прогресс
  cancelTask(taskId: string): Promise<boolean>;                            // отмена
  getResult(taskId: string): Promise<Blob>;                                // результат
  onTaskStatusChange(callback: (e: TaskProgress) => void): () => void;    // подписка
}
```

## ✅ Что реализовано

1. **Асинхронная очередь задач** — максимум 1 задача одновременно (для стабильности браузерного ML)
2. **Таймаут 30 секунд** — автоматическая отмена при превышении
3. **Поддержка HEIC** — через heic2any (WASM)
4. **EXIF Orientation** — корректный поворот изображений
5. **WebGL-шейдеры** — аппаратное ускорение фильтров
6. **2D-фоллбэк** — при отсутствии WebGL
7. **Демо-интерфейс** — чат-подобный UI с прогресс-баром
8. **Тест-датасет** — кнопка для обработки 35 плиток из спрайта

## ⚠️ Ограничения / что можно улучшить

- ONNX-модель **не включена** в поставку (отключена по умолчанию для стабильности). Для полноценного ML нужно добавить обученную `.onnx` модель и включить `VITE_ENABLE_ONNX=true`
- Готовое изображение ограничено 800x600 в encoder (можно увеличить)
- Для 15Мп изображений на слабых GPU возможно автоматическое даунскейлирование
