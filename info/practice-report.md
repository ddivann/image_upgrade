# Отчёт обучающегося по практике

## 1. Введение

В рамках практики выполнены работы по созданию прототипа системы улучшения изображений, работающей **в браузере пользователя**. Улучшение изображения предусматривает корректировку следующих параметров: **яркость**, **контрастность**, **цветность (насыщенность)**.

Работа выполнялась в соответствии с индивидуальным заданием и планом этапов (см. `info/stages`). Основная идея решения — обработка выполняется на стороне пользователя, без передачи исходных изображений на внешний сервер.

## 2. Цель и исходные требования

### 2.1. Цель

Цель практики — разработать прототип программного решения, который принимает изображение, выполняет его обработку в асинхронном режиме и возвращает пользователю результат, а также предоставляет возможность отслеживания хода выполнения и отмены операции.

### 2.2. Основные требования (по заданию)

- работа в массовых современных браузерах;
- суммарный объём поставляемого кода — до 10 МБ;
- обработка изображений до 15 Мп;
- ограничение времени обработки: до 30 секунд (максимум), целевой средний показатель — около 5 секунд;
- поддержка форматов: JPG, PNG, HEIC, BMP;
- асинхронная работа без блокировки пользовательского интерфейса.

## 3. Использованные средства

В ходе практики применялись следующие средства разработки (в объёме, достаточном для прототипа):

- TypeScript;
- Vite (сборка и подготовка проекта);
- Web Worker (выполнение обработки вне основного потока);
- стандартные браузерные средства обработки изображений (Canvas/OffscreenCanvas, createImageBitmap);
- библиотека heic2any для обработки изображений формата HEIC.

## 4. Краткое описание реализованного решения

### 4.1. Общая схема работы

Решение построено на принципе «задача → обработка → результат»:

- пользователь передаёт изображение в систему;
- создаётся задача на обработку, ей присваивается идентификатор;
- обработка выполняется в фоновом режиме;
- по мере выполнения обновляется статус и процент готовности;
- после завершения возвращается результат (файл с обработанным изображением).

### 4.2. Управление задачами

Реализован интерфейс для управления задачами:

- постановка задачи на обработку;
- получение текущего статуса и прогресса;
- отмена задачи пользователем;
- получение результата обработки.

Также предусмотрены очередь задач и ограничение на параллельное выполнение (для стабильности работы прототипа), а также таймаут до 30 секунд.

### 4.3. Этапы обработки

Для удобства контроля выполнения обработка разбита на стадии:

- декодирование изображения;
- анализ/подбор параметров;
- применение улучшения;
- кодирование результата.

Часть функциональности (автоматический подбор параметров на базе ML и ускоренная обработка на GPU) в текущем прототипе реализована в упрощённом виде. Это сделано для фиксации структуры и проверки общей работоспособности.

### 4.4. Демонстрация работы

Подготовлена демонстрационная страница, на которой можно:

- загрузить изображение;
- наблюдать прогресс выполнения;
- отменить обработку;
- получить и просмотреть результат.

## 5. Проверка работоспособности

В ходе практики выполнены базовые проверки:

- корректная постановка задач и получение идентификатора;
- обновление статуса и прогресса по стадиям;
- получение результата по завершении;
- отмена обработки по запросу пользователя;
- обработка ошибок и корректное завершение выполнения.

Отдельно отмечено, что для полного соответствия требованиям по изображениям 15 Мп и целевому времени обработки требуется дальнейшая оптимизация и расширенное тестирование в разных браузерах (это предусмотрено планом работ).

## 6. Заключение

По итогам практики подготовлен работающий прототип системы улучшения изображений в браузере пользователя. Реализованы основные механизмы: асинхронная обработка, управление задачами, отображение прогресса, отмена и получение результата, а также поддержка основных форматов (включая HEIC через конвертацию).

Результаты работы соответствуют индивидуальному заданию в части архитектуры и функционального прототипа. Дальнейшее развитие проекта предполагает доведение алгоритмов улучшения до полноценной версии (в т.ч. ML‑подбор параметров и ускорение обработки), оптимизацию производительности и расширение кросс‑браузерной валидации.
